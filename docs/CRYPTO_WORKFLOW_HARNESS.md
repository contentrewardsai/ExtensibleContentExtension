# Content-player workflow harness (future)

**Goal:** Drive **multi-step** workflow JSON through the same **content player** path users run (not only isolated **`CFS_*`** messages), with **devnet** or **Anvil fork** RPC and an **unlocked** automation wallet.

## Why it is separate

Message-level and side-panel smokes catch **payload → service worker → signer** bugs for one operation. They do **not** routinely catch:

- Step-to-step **variable** / row merge issues
- **Retry** and **orchestration** logic in the player
- **Storage** and **session** interactions across steps

## Implementation sketch (backlog)

- Fixture: small workflow JSON + CSV rows (or inline rows) checked into **`test/e2e/`** or generated by a build script.
- Driver: Playwright opens the side panel + fixture page, ensures wallets (**`CFS_CRYPTO_TEST_ENSURE_WALLETS`** or profile seed), runs **RUN_WORKFLOW** (or UI equivalent), asserts DOM / storage / messages.
- **Env gates:** Same pattern as **`E2E_CRYPTO_*`** — off by default; document RPC and fork requirements.

This repo does **not** yet ship such a harness; treat it as **large-scope** work when workflow regressions justify the maintenance cost.
