# Solana automation (GitHub-safe)

For **BNB Smart Chain / PancakeSwap** automation, see **docs/BSC_AUTOMATION.md** (steps **`bscPancake`** / **`bscQuery`**, `npm run build:evm`).

This extension can run **automated** Solana actions using a **hot keypair** stored in **Chrome extension storage**:

- **Solana Jupiter swap** — routed swaps (optional `dexes` / `excludeDexes` / direct routes; optional **prioritization fee** lamports or `auto`; optional **dynamic compute unit limit** toggle; optional **`jupiterWrapAndUnwrapSol`** default **true** — set **false** on **solanaJupiterSwap** / Pump-or-Jupiter Jupiter branches when using **solanaWrapSol** + WSOL mint so Jupiter omits redundant wrap/unwrap). Step fields mirror `CFS_SOLANA_EXECUTE_SWAP` in `background/solana-swap.js`.
- **Solana transfer SOL** — native `SystemProgram.transfer` (lamports); **SPL / Token-2022** — **`solanaTransferSpl`** / **`CFS_SOLANA_TRANSFER_SPL`** (`TransferChecked` from the automation wallet’s ATA; optional idempotent recipient ATA creation). Both support **optional compute budget** fields (`computeUnitLimit`, `computeUnitPriceMicroLamports`). Rebuild `npm run build:solana` after web3 / SPL bundle changes.
- **Prep / WSOL / read-only** — **`solanaEnsureTokenAccount`** (`CFS_SOLANA_ENSURE_TOKEN_ACCOUNT`): idempotent ATA create for a mint, skipped if the ATA exists; optional **`additionalMints`** runs the same ensure for more mints in sequence, with optional **`saveEnsureResultsVariable`** JSON for all results. **`solanaWrapSol`** / **`solanaUnwrapSol`** (`CFS_SOLANA_WRAP_SOL` / `CFS_SOLANA_UNWRAP_WSOL`): wrap native SOL into WSOL or close the WSOL ATA without Jupiter. **`solanaReadBalances`** / **`solanaReadMint`** (optional **`includeMetaplexMetadata`** on **`mintInfo`**: **`getMint`** + Metaplex PDA in parallel; optional **`fetchMetaplexUriBody`** on **`mintInfo`** requires **`includeMetaplexMetadata`** and uses the same URI fetch as **`metaplexMetadata`**) / **`solanaReadMetaplexMetadata`** (`CFS_SOLANA_RPC_READ` with `nativeBalance`, `tokenBalance`, `mintInfo`, or **`metaplexMetadata`**): read-only RPC for lamports, SPL balance at the derived ATA, SPL mint fields (including **mint authority**), or Metaplex Token Metadata PDA (**name** / **symbol** / **uri**); optional **`fetchMetaplexUriBody`** HTTPS-gets **`uri`** (capped, blocked private hosts; **`ipfs://`** / **`ipns://`** / **`ar://`** via configurable gateways; defaults ipfs.io + arweave.net). Response includes **`uriResolvedForFetch`** (final HTTPS URL) when fetch is requested. Default **owner** for balance reads is the **Primary** automation wallet’s public key from **`cfs_solana_wallets_v2`** (legacy: **`cfs_solana_public_key_hint`** after one-time migration).
- **Solana Pump.fun buy / sell** — bonding-curve trades via `@pump-fun/pump-sdk` (bundled in `background/pump-sdk.bundle.js`). Graduated tokens are rejected; use Jupiter for those. After upgrading the SDK, run `npm run build:pump` and reload the extension.
- **Pump / Raydium probe** — `CFS_PUMPFUN_MARKET_PROBE` + step `solanaPumpMarketProbe` (bonding-curve state + optional Raydium v3 pool-by-mint check on mainnet). **`solanaPumpOrJupiterBuy`** / **`solanaPumpOrJupiterSell`** probe then trade on Pump curve or Jupiter (buy: WSOL→mint; sell: mint→WSOL). On the **Jupiter** branch they accept the same optional **`jupiterPrioritizationFeeLamports`** / **`jupiterDynamicComputeUnitLimit`** / **`jupiterWrapAndUnwrapSol`** fields as **`solanaJupiterSwap`**. Those composite steps can also **write probe fields to row variables** (same optional names as `solanaPumpMarketProbe` when configured: Raydium check/count/spot-found, bonding-curve flags, optional Pump error and Raydium detail) for `runIf` without a separate probe step. Per-step field lists live under **steps/solanaPumpOrJupiterBuy/README.md** and **steps/solanaPumpOrJupiterSell/README.md**. **Gates:** `requireRaydiumPoolForPump` (fail Pump path unless Raydium reports a pool for mint/quote on mainnet) and `skipPumpIfRaydiumPoolFound` (force Jupiter when a Raydium pool exists, even if the bonding curve is still readable). **Finer customization:** chain **`solanaPumpMarketProbe`** → **`runIf`** / **`rowMath`** / **`llm`** → **`solanaPumpfunBuy`** or **`solanaJupiterSwap`** (or sell equivalents) instead of the composite steps — see **steps/solanaPumpOrJupiterBuy/README.md** § Multi-step alternative.
- **Sellability probe** — **`solanaSellabilityProbe`** + **`CFS_SOLANA_SELLABILITY_PROBE`** (**`background/solana-sellability-probe.js`**): spends **`solLamports`** or **`spendUsdApprox`** (Jupiter public SOL price), buys then immediately sells the received token balance (same Pump vs Jupiter routing as the composite buy/sell steps). Optional **`jupiterCrossCheckMaxDeviationBps`** / **`jupiterCrossCheckOptional`** on Jupiter legs match **`solanaJupiterSwap`**. Row vars for ok / venue / both signatures / amounts. See **steps/solanaSellabilityProbe/README.md**.
- **Meteora DLMM** ([pools UI](https://www.meteora.ag/pools)) — **`meteoraDlmmAddLiquidity`** / **`meteoraDlmmRemoveLiquidity`** / **`meteoraDlmmClaimRewards`** → `CFS_METEORA_DLMM_*` in **`background/meteora-dlmm.js`**, SDK bundle **`npm run build:meteora`** (`@meteora-ag/dlmm`). Covers DLMM **LB pair** pools (add creates a **new** position; save **position** pubkey for remove/claim). Use **LB pair** addresses only for these steps.
- **Meteora DAMM v2 / CP-AMM** (same [pools UI](https://www.meteora.ag/pools)) — **`meteoraCpammSwap`** (exact-in) / **`meteoraCpammQuoteSwap`** (read-only exact-in quote, `CFS_METEORA_CPAMM_QUOTE_SWAP`, no keypair) / **`meteoraCpammSwapExactOut`** (exact-out via **`getQuote2` + `swap2`**, `CFS_METEORA_CPAMM_SWAP_EXACT_OUT`) / **`meteoraCpammQuoteSwapExactOut`** (read-only exact-out quote, `CFS_METEORA_CPAMM_QUOTE_SWAP_EXACT_OUT`) / **`meteoraCpammAddLiquidity`** / **`meteoraCpammDecreaseLiquidity`** / **`meteoraCpammRemoveLiquidity`** / **`meteoraCpammClaimFees`** / **`meteoraCpammClaimReward`** → `CFS_METEORA_CPAMM_*` in **`background/meteora-cpamm.js`**, SDK bundle **`npm run build:meteora-cpamm`** (`@meteora-ag/cp-amm-sdk`). On-chain steps (not read-only quotes) accept optional **`computeUnitLimit`** / **`computeUnitPriceMicroLamports`** like **`solanaTransferSol`**, using the shared **`__CFS_try_parse_compute_budget_instructions`** helper from **`solana-swap.js`** (prepends compute-budget instructions; needs Solana bundle with `ComputeBudgetProgram`). **Pool** is the CP-AMM pool account (not DLMM). Exact-in **swap** / **quote** **`getQuote`** loads both mints and **`getEpochInfo`** so **inputTokenInfo** / **outputTokenInfo** include Token-2022 transfer-fee math when present. Exact-in **swap** uses **amountInRaw**; optional **`minimumAmountOutRaw`** raises the on-chain min out to **max**(quoted min, floor), e.g. after **meteoraCpammQuoteSwap**. Exact-out **swap** / **quote** use **amountOutRaw**; optional **`maximumAmountInRaw`** lowers the on-chain max input to **min**(quoted max, ceiling), e.g. after **meteoraCpammQuoteSwapExactOut**. **Add** creates or **increases** a position. **Decrease** removes a **fraction** of liquidity (**removeLiquidityBps** 1–10000) without closing. **Remove** drains, claims fees, and closes. **Claim fees** vs **claim incentive** (`rewardIndex` 0|1) are separate. Position actions require the automation wallet to be the on-chain **owner**.
- **Raydium Standard AMM** — **Liquidity:** `raydiumAddLiquidity` / `raydiumRemoveLiquidity` (OpenBook `marketId` pools only). **Single-hop swap:** `raydiumSwapStandard` + `CFS_RAYDIUM_SWAP_STANDARD` (`background/raydium-standard-swap.js`).
- **Raydium CPMM** — `raydiumCpmmAddLiquidity` / `raydiumCpmmRemoveLiquidity` + `CFS_RAYDIUM_CPMM_*` (`background/raydium-cpmm-liquidity.js`): cp-swap style pools; same SDK bundle (`npm run build:raydium`).
- **Raydium CLMM** — **Swap:** **`raydiumClmmSwap`** (`CFS_RAYDIUM_CLMM_SWAP_BASE_IN`, fixed input) and **`raydiumClmmSwapBaseOut`** (`CFS_RAYDIUM_CLMM_SWAP_BASE_OUT`, exact output) in `background/raydium-clmm-swap.js` (`PoolUtils` in the Raydium bundle). **Read-only quotes** (no tx): workflow steps **`raydiumClmmQuoteBaseIn`** / **`raydiumClmmQuoteBaseOut`** → `CFS_RAYDIUM_CLMM_QUOTE_BASE_IN` / `CFS_RAYDIUM_CLMM_QUOTE_BASE_OUT` (same payload shape as the matching swap; wallet must still be unlocked for RPC pool fetch). **Liquidity / positions** in `background/raydium-clmm-liquidity.js`: open (**`raydiumClmmOpenPosition`** / **`raydiumClmmOpenPositionFromLiquidity`**); rewards (**`raydiumClmmCollectReward`**, **`raydiumClmmCollectRewards`**, **`raydiumClmmHarvestLockPosition`**); **lock** **`raydiumClmmLockPosition`** → then harvest with the lock NFT; add (**`raydiumClmmIncreasePosition`** / **`raydiumClmmIncreasePositionFromLiquidity`**); **decrease** **`raydiumClmmDecreaseLiquidity`**; **close** empty positions **`raydiumClmmClosePosition`** (liquidity must be zero). Batch rewards: one tx per mint until failure. Raydium may disable features by region (`availability`).
- **Perpetuals** — on-chain **order execution** not implemented; see **docs/PERPS_SPIKES.md**, messages **`CFS_PERPS_AUTOMATION_STATUS`** and read-only **`CFS_JUPITER_PERPS_MARKETS`** (Jupiter API key), and step **`solanaPerpsStatus`** (row vars + optional markets JSON).

## Pulse (Following): watch addresses and optional Following automation

This is **not investment advice**. Copying activity observed on-chain does **not** guarantee the same outcome. **Latency** (RPC polling, quote time) and **MEV** mean your fills and ordering will differ from the target’s.

- **Profile wallets** are **user-managed** in Pulse. Attaching an address to someone’s profile is **not** cryptographic proof that they control that address.
- **Watch transport** uses the service worker (`background/solana-watch.js`), alarm **`cfs_solana_watch_poll`** (minimum **1** minute). **HTTPS JSON-RPC** uses `getSignaturesForAddress` / `getTransaction` against **`resolveWatchRpcUrl`**: optional **`cfs_solana_watch_rpc_url`**, else **Helius** (`cfs_solana_watch_helius_api_key` → `https://mainnet.helius-rpc.com/?api-key=…` or devnet equivalent), else the same URL as **Settings → Solana → RPC URL** / public default. **WebSocket** (`logsSubscribe` per watched address): optional **`cfs_solana_watch_ws_url`**, or **wss** derived from the Helius key when the WS field is empty (mainnet / devnet). When a WS URL resolves and subscriptions succeed, that tick usually **skips HTTP polling** (push path); **every 6th** alarm tick still runs an **HTTP poll** to reconcile cursors and recover missed `logsNotification` events after service-worker restarts. If WS setup fails, the worker **falls back to HTTP** every tick. Jitter, shuffle, retry, and **`cfsSolanaWatchLastPoll`** (`reason` includes **`polled`** vs **`websocket`**) behave as before. Configure keys/URLs under **Settings → Solana automation → Pulse / Following — Solana watch**. A backend indexer remains a future option — see **docs/PULSE_INDEXER_RFC.md** for a design sketch (not implemented).
- When you save Following data, the sidepanel updates **`cfsPulseSolanaWatchBundle`** in **`chrome.storage.local`** so the worker knows which Solana addresses to poll. **Automation policy** (mode, sizing, paper, Jupiter wrap) is configured on **always-on workflows** via **`workflow.followingAutomation`** and the **`selectFollowingAccount`** bind step; see **`docs/FOLLOWING_AUTOMATION_PIPELINE.md`**.
- **Workflow gate (Library):** Pulse Following does not poll or automate unless **`chrome.storage.local.workflows`** has at least one workflow. **Legacy behavior:** if no workflow sets **`alwaysOn.enabled`**, that is enough (any Library workflow). **Strict behavior:** if **any** workflow has **`alwaysOn.enabled`**, the service worker unions **`alwaysOn.scopes`** (Solana watch, BSC watch, Following automation per chain) from those workflows and evaluates **`alwaysOn.conditions`**; Solana watch and automation run only when the merged result allows it (`shared/cfs-always-on-automation.js`, **`__CFS_evaluateFollowingAutomation`**). Configure under **Library → Background automation (Following)** on the selected workflow.
- **Execution** always builds a **new** swap for **your** Solana automation wallet via **Jupiter** (`__CFS_solana_executeSwap`); the extension **never** replays the target’s serialized transaction. **Jupiter wrap/unwrap SOL** is controlled by **`workflow.followingAutomation.jupiterWrapAndUnwrapSol`** when a workflow is bound; unbound flows may still read **`cfsFollowingAutomationGlobal.jupiterWrapAndUnwrapSol`** when no workflow matches.
- **Sizing modes** (workflow **`followingAutomation`**): **off**, **proportional**, **fixed token (raw)**, **fixed USD** (Jupiter public price APIs; skip **`price_unavailable`** if missing). Thin-market prices can be stale — not an oracle guarantee.
- **Price drift / tx age (workflows):** Pulse no longer applies global drift, staleness, or cooldown gates in storage. Use **`solanaWatchRefresh`** → **`solanaWatchReadActivity`** → **`watchActivityFilterTxAge`** / **`watchActivityFilterPriceDrift`** (and swap steps) to filter watch **`activity`** JSON the same way. Enriched **`swap_like`** rows include mints, **`targetPrice`**, leg raws, and **`targetBlockTimeUnix`** for those steps.
- **Global token blocklist (Settings):** **`cfsFollowingAutomationGlobal.globalTokenBlocklist`** — **`solana`** (mint base58 per line) and **`evm`** (`0x` per line). Canonical **WSOL**, **WBNB**, and **zero** address cannot be saved. If a swap touches a listed mint/token, automation skips with **`mint_denylisted`** / **`token_denylisted`**. Implemented in **`shared/cfs-global-token-blocklist.js`**.
- **Paper mode:** Prefer **`workflow.followingAutomation.paperMode`** on a bound always-on workflow. Unbound flows may still honor **`cfsFollowingAutomationGlobal.paperMode`** when no workflow matches.
- **Watch bundle size:** polling walks every bundle address each tick; with **many** watched addresses, public RPCs may **rate-limit**—the Pulse bundle line warns when there are more than **12** entries.
- **Auto-exec** defaults to **off**. With auto-exec disabled, the worker may show a **browser notification** when an automation signal is detected (requires the **notifications** permission). Duplicate notifications for the **same watched tx** (`walletId` + signature) are suppressed for **~90s**; the in-memory dedupe map is **trimmed** if it grows past a few hundred keys. With auto-exec **on**, the automation wallet must be **unlocked** in Settings; use a **low-balance** hot wallet.
- **Swap classification (heuristic):** simple two-leg balance changes are treated as **`swap_like`** when the quote leg is **wrapped SOL (WSOL)** or **mainnet USDC** (`EPjF…t1v`). Native SOL-only legs (no WSOL token row) are still mapped to a WSOL quote for buy/sell hints. Multi-hop or exotic quotes stay **`unknown`**.
- **Debug / UI hooks:** messages **`CFS_SOLANA_WATCH_GET_ACTIVITY`** (optional **`limit`**), **`CFS_SOLANA_WATCH_REFRESH_NOW`** (optional **`skipJitter: true`** so manual refresh is immediate), and **`CFS_SOLANA_WATCH_CLEAR_ACTIVITY`** (empties the activity list). **`CFS_FOLLOWING_AUTOMATION_STATUS`** returns **`__CFS_evaluateFollowingAutomation`** output (workflow gate + always-on scopes) for Pulse UI. Workflow steps **`solanaWatchReadActivity`** / **`solanaWatchRefresh`** call the same messages. Pulse **Export JSON** includes the last **100** activity rows plus snapshots of **`cfsSolanaWatchLastPoll`**, watch bundle summary (**`updatedAt`**, **`entryCount`**), **`cfs_solana_cluster`**, and a **JSON-cloned** **`cfsFollowingAutomationGlobal`** (for debugging; no automation private keys). Recent events are stored under **`cfsSolanaWatchActivity`** (duplicate **signature + watched address** rows are not appended again). Each row records **`solanaCluster`** (from **`cfs_solana_cluster`**) so Pulse can link signatures to **Solscan** (mainnet vs devnet); older rows without that field use the current **Settings → cluster** value for links. The Pulse tab shows a **banner** when global **watch** or **automation** pause is on, lists recent events with short **automation outcome** labels (including a truncated **`exec_fail`** detail when present), and offers **Refresh poll** and **Clear activity**. **`cfsFollowingAutomationGlobal`** holds **watchPaused**, **automationPaused**, and **globalTokenBlocklist**. When **`watchPaused`** is true, the worker **does not RPC-poll** watched addresses (alarms still fire; tick returns idle). When **`automationPaused`** is true, the worker skips **notifications and execution** for Following automation (watch polling continues unless **`watchPaused`**).

**Background message index:** the comment at the top of **`background/solana-swap.js`** lists **CFS_*** message types and their **`background/*.js`** modules (Jupiter, transfers, Pump, Raydium, **Meteora DLMM**, **Meteora CP-AMM** including exact-out swap/quote, CLMM, perps, wallet). **`background/solana-watch.js`** documents watch and Following automation storage keys and the **`CFS_SOLANA_WATCH_*`** messages. Payload shapes are spelled out in each handler file’s header (e.g. **`background/meteora-cpamm.js`**). Service worker **`[CFS_CRYPTO]`** rate-limit logs: **docs/CRYPTO_OBSERVABILITY.md**.

**Per-step docs:** Pump/Raydium/Jupiter automation steps each have a **steps/{id}/README.md** (configuration, background messages, row variables). See **steps/README.md** § Step-specific documentation for the index.

Nothing secret belongs in the git repository.

## Do not commit

- Private keys, mnemonics (12/24 words), or `id.json` / keypair JSON files
- `.env` files with RPC API keys (already covered by `.gitignore`)
- Workflow JSON that embeds secrets (workflows should only reference **public** mints, amounts, slippage, and row variable names)
- A filled-in `config/solana-local.json` (see `config/solana-local.example.json` for shape only)

### Custom RPC (mainnet / devnet / private endpoints)

If **RPC URL** is empty, steps use the public defaults (**`api.mainnet-beta.solana.com`** or **`api.devnet.solana.com`**, also listed under **`host_permissions`** in **`manifest.json`**). For **Helius, QuickNode, Alchemy, localhost tunneled to HTTPS**, or any other JSON-RPC URL, paste it into **RPC URL** in Settings. The manifest includes **`<all_urls>`**, so the service worker can call that endpoint without adding a new permission entry for each provider.

### Password encryption

- **Encrypt on import** or **Encrypt existing plaintext** wallets store per-wallet **`encJson`** blobs; **one vault password** decrypts every encrypted wallet on **Unlock** (session map). Automated workflow runs need **Unlock** when the **Primary** wallet is encrypted.
- **Forgotten password:** re-import from an offline backup of the private key or mnemonic.

## CI / automated tests

The **Extension checks** workflow (`.github/workflows/extension-checks.yml`) runs **`npm run build:step-tests`** and fails if **`test/unit-tests.html`** or **`settings/settings.html`** drift from **`steps/manifest.json`** (commit regenerated files after adding or reordering steps). It also runs **`validate:steps`** (every **`steps/{id}/step.json`** plus **`handler.js`** / **`sidepanel.js`**), **`test:solana`**, **`test:evm-bundle`**, **`test:bsc-watch-wired`**, **`check:content-bundle`**, **`test:unit`**, **`test:crypto-manifest-hosts`**, **`test:crypto-observability-wired`**, **`test:cfs-e2e-testids-wired`** (Playwright **`data-testid`** hooks vs **`test/e2e/cfs-e2e-testids.mjs`**), and **`test:apify`**. Optional read-only RPC smoke: **docs/CRYPTO_CI_SMOKE.md**. Bundle upgrades after npm bumps: **docs/CRYPTO_BUNDLE_UPGRADE_RUNBOOK.md**.

Use **GitHub Actions secrets** (or another secret store) to inject throwaway devnet keys into a test harness if you add on-chain tests. **`npm run test:solana`** runs all Solana-related service worker bundle checks (web3 + Pump + Raydium + Meteora DLMM + Meteora CP-AMM). You can also run each verify script separately (`test:solana-bundle`, `test:pump-bundle`, etc.). **`npm run validate:steps`** validates every **`steps/{id}/step.json`** (including Solana step definitions). **`npm run test:unit`** includes handler payload tests for workflow steps. None of these hit RPC or sign transactions. Do not commit test keys into the repo.

## Risk summary

Automated signing means anyone with access to this browser profile or a malicious build of the extension could move funds. Password encryption reduces **at-rest** exposure on disk but does not remove **session** risk while unlocked. Use a **dedicated low-balance wallet** for automation. Back up the private key or mnemonic **outside** the browser; clearing extension data without a backup **strands** access to that local copy (on-chain funds remain if you have the key elsewhere).

**Pulse Following automation** adds **counterparty and model risk**: misclassified transactions, illiquid mints, and failed swaps. Treat watch and automation as **experimental** and monitor balances.
