# asterFuturesAccount

Signed USER_DATA reads via **`CFS_ASTER_FUTURES`** (`asterCategory: account`). Configure **API key + secret** in **Settings → Aster futures API** (not in workflow JSON).

Operations include `balance`, `account`, `positionRisk`, `openOrders`, `allOrders`, `queryOrder`, `historicalTrades`, `userTrades`, `income`, `commissionRate`, `adlQuantile`, `forceOrders`, `leverageBracket`, `getPositionMode`, `getMultiAssetsMargin`.

See [INTEGRATIONS.md](../../docs/INTEGRATIONS.md).
